                         CHIP MEMORY MAP DESCRIPTION

================================================================================
  FLASH MEMORY (Physical: 2Mb built-in flash)
================================================================================

  Logical Address Range          Physical Address         Description
  ─────────────────────────────────────────────────────────────────────────────
  0x00000000 - 0x0000FFFF        0x00000000 (built-in)   Bootloader (ROM)
                                                          ARM interrupt vector table
                                                          RF + MAC

  0x00010000 - 0x0012FFFF        0x00011000               Application (app)

  0x00130000 - 0x001FFFFF        —                        OTA


================================================================================
  RAM — BLOCK 1 (main)
================================================================================

  0x00400000 - 0x0043FFFF        RAM (main firmware RAM)

  ─────────────────────────────────────────────────────────────────────────────
  Interrupt Hook Table  [0x00400000 - 0x0040001F]  (32 bytes / 0x20)
  ─────────────────────────────────────────────────────────────────────────────

    The bootloader (ARM architecture) uses the first 0x20 bytes of RAM as
    a software hook table for ARM exception vectors. The main firmware controls
    interrupt handling by writing handler addresses into these hooks.

    Offset    Hook Name          Description
    ────────  ─────────────────  ────────────────────────────────────────────
    +0x00     hook_IRQ           IRQ (Interrupt Request) handler hook
    +0x04     hook_FIQ           FIQ (Fast Interrupt Request) handler hook
    +0x08     hook_SWI           SWI (Software Interrupt) handler hook
    +0x0C     hook_undefined     Undefined instruction handler hook
    +0x10     hook_pabort        Prefetch Abort handler hook
    +0x14     hook_dabort        Data Abort handler hook
    +0x18     hook_reserved      Reserved exception hook
    +0x1C     (nothing)          Unused / padding

  ─────────────────────────────────────────────────────────────────────────────
  General RAM  [0x00400020 - 0x0043FFFF]
  ─────────────────────────────────────────────────────────────────────────────

    General-purpose RAM used by the main firmware.

  0x00440000 - 0x007FFFFF        RAM mirrors
                                 (mirrors of the 0x00400000 RAM block)


================================================================================
  PERIPHERALS
================================================================================

  0x00800000 - 0x008FFFFF        Peripherals (exact layout unknown)

================================================================================
  RAM — BLOCK 2
================================================================================

  0x00900000 - 0x0093FFFF        RAM Block 2
                                 - Empty state filled with 0xAA
                                 - Purpose unclear; possibly not used by
                                   main firmware (e.g. dedicated to RF/MAC
                                   subsystem or hardware coprocessor)

================================================================================
  HARDWARE IP REGISTERS / PERIPHERALS (upper address space)
================================================================================

  Address         Block           Description
  ──────────────  ──────────────  ──────────────────────────────────────────
  0x01000000      AGC             Automatic Gain Control
                  MDM CFG         Modem configuration registers
                  MDM STAT        Modem status registers
                  ?               Unknown registers

  0x01050000      RC              (?)

  0x10A00000      DMA             (?)

  0x10910000      INTC            Interrupt Controller

  0x10E00000      LA              Unknown (?)

  0xC0000000      NXMAC HW        NX MAC hardware core (802.11 MAC core)
                  MAC CORE

  0xC0008000      MAC PL          MAC Physical Layer interface (?)


================================================================================
  SUMMARY — ADDRESS SPACE OVERVIEW
================================================================================

  0x00000000 - 0x0000FFFF   Flash: ARM vectors + Bootloader + RF/MAC
  0x00010000 - 0x0012FFFF   Flash: Application
  0x00130000 - 0x001FFFFF   Flash: OTA
  0x00200000 - 0x003FFFFF   Unknown / Unreadable (with 0xFF and 0xAA regions)
  0x00400000 - 0x0043FFFF   RAM Block 1 (hook table @ 0x400000 + 0x20 bytes)
  0x00440000 - 0x007FFFFF   RAM Block 1 mirrors
  0x00800000 - 0x008FFFFF   Peripherals
  0x00900000 - 0x0093FFFF   RAM Block 2 (empty = 0xAA)
