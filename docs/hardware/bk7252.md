# BK7252 Chip Reference

BK7252 is a 2.4 GHz 802.11b/g/n + BLE 4.2 Wi-Fi Audio/Video SoC (Beken
Corporation, 2019). The MCU core is a 32-bit ARM968E-S running up to 180 MHz.
512 KiB internal DTCM RAM. Package: QFN68, 8 × 8 mm (part number BK7252QN68).

Peripheral bus base `0x00800000`. Memory map in `docs/memory_map.md`.

## Clock Sources

Seven root clocks:

| Name   | Source                  | Frequency             | Notes                                      |
| ------ | ----------------------- | --------------------- | ------------------------------------------ |
| X26M   | External XTAL           | 26 MHz (typ)          | Reference for DPLL and I2SPLL; tunable load cap 6–18 pF in 64 steps; ~1 ms startup |
| DCO    | Internal DCO            | 26–180 MHz            | ±2 % after calibration; few µs startup    |
| X32K   | External XTAL           | 32.768 kHz            | Low-power domain clock                     |
| D32K   | Divided from X26M       | 32 kHz                | Software-generated low-power clock         |
| ROSC   | Internal ring oscillator | ~32 kHz              | ±2 % after calibration                    |
| DPLL   | PLL (ref: X26M)         | 480 MHz               | High-speed PLL                             |
| I2SPLL | PLL (ref: X26M)         | 1024 × FS (44.1/48 kHz) | Audio PLL; MCLK = I2SPLL ÷ 1/2/4/8/16  |

**LPO_CLK** (low-power clock) is selected from X32K, D32K, or ROSC.

**PCLK for CIS/MJPEG** is divided from DPLL; typical values 24 MHz, 48 MHz, 96 MHz.

## Reset

Power-on reset, digital power-on reset, and watchdog reset are equivalent: all
reset the full chip to initial state. The always-on domain (32-bit RTC timer +
16-bit retention register) is not affected by these resets — it resets only on
system power-on reset (VBAT removal).

Waking from shutdown or deep sleep re-asserts the full system reset sequence.

---

## Power Management

Four low-power modes, in order of decreasing power:

| Mode               | MCU | Peripherals | Always-on | Wake Sources               | I_typ |
| ------------------ | --- | ----------- | --------- | -------------------------- | ----- |
| Normal Standby     | off | clocked     | on        | Any peripheral interrupt   | 100 µA |
| Low Voltage Standby | off | low-V      | on        | GPIO edge, RTC timer       | 40 µA  |
| Deep Sleep         | off | off         | on        | GPIO edge, RTC timer       | 10 µA  |
| Shutdown (CEN = 0) | off | off         | off       | CEN high, GPIO13           | 1 µA   |

- **Deep sleep** preserves the retention registers. GPIO states are retained.
- **Low Voltage Standby** reduces supply to a retention level; only GPIO or RTC
  can resume to full operating voltage.
- **Shutdown** via software is also possible (in addition to hardware CEN = 0).
- Waking from shutdown or deep sleep triggers a full chip reset.

## Peripheral Capabilities

### UART
Two sets. Maximum baud rate 6 Mbps. Data width 5–8 bits; parity even/odd/none;
stop bits 1 or 2. UART1 supports hardware RTS/CTS flow control. A continuously
low level on TX or RX wakes the MCU from Low Voltage Standby.

### SPI
- High-speed SPI: slave-only, up to 50 MHz, dedicated DMA channel.
- Low-speed SPI: master or slave, up to 8 MHz; MSB or LSB first; latch on
  rising or falling clock edge.

### SDIO
Master or slave. 1-bit to 4-bit mode. Up to 50 MHz. Dedicated DMA channel.

### I2C
Two sets. 400 kHz, 7-bit addressing. Generates interrupt on SCL low timeout or
bus-idle duration exceeding a programmable threshold.

### USB
Full-speed USB 2.0 (12 Mbps). Host and device; mode selected by software
register. Dedicated DMA channel.

### General-Purpose ADC
Multi-channel. 10-bit high-speed or 16-bit low-speed with internal decimation
filter. Modes: single-shot, continuous, software-triggered. Sources: GPIO pins
(ADC1–7), VSYS voltage, temperature sensor, internal calibration reference.

### PWM
Six 32-bit channels (PWM0–5). Clock source: high-speed or LPO_CLK, selectable
per channel. Capture mode: counts cycles between two rising or two falling edges.

### Timer
Two groups of three 32-bit timers each. Group 1 clocked from X26M with 4-bit
prescaler. Group 2 clocked from LPO_CLK with 4-bit prescaler. Watchdog timer
(LPO_CLK; stops if MCU stops or powers off). Always-on timer (LPO_CLK domain;
runs even when MCU is off).

### I2S
Master or slave. Sample rate 8–96 kHz. MCLK output available on GPIO21.
Supports PCM frame format.

### IrDA
Hardware IrDA decoder plus capture-timer mode for software decoding.

### QSPI
Shared data lines (IO0–IO3), separate clock and chip-select for FLASH and RAM.
Up to 120 MHz. FLASH and RAM clock rates can differ.

### CMOS Sensor Interface / MJPEG
8-bit DVP interface: MCLK (GPIO27), PCLK (GPIO29), HSYNC (GPIO30), VSYNC
(GPIO31), data D0–D7 (GPIO32–GPIO39). Clocked from DPLL at 96 MHz; sensor MCLK
is integer-divided from 96 MHz. Supported sensors: OV7676, OV7670, GC0308,
GC0309, GC0329, PAS6329. YUV input formats: YUYV, UYVY, YYUV, UVYY (HSYNC and
VSYNC polarity independently configurable). Hardware VGA-grade MJPEG encoder;
output written directly to RAM by dedicated DMA.

### Security
- 32-byte eFUSE: bytes 0–15 = code encryption; bytes 16–30 = user data (e.g.
  MAC address); byte 31 = JTAG interface disable. eFUSE written via GPIO20–GPIO23
  at download mode. Bytes 0–15 cannot be read via download port.
- True random number generator.
- Hardware AES/RSA/SHA accelerator: AES up to AES-256 with GCM mode, RSA4096,
  SHA-512.

### Audio Peripheral
16-bit stereo DAC (two channels) + 16-bit ADC (one channel). Microphone input
amplifier (0–42 dB, 2 dB step). Line-in input with 0–6 dB amplifier (2 dB
step). Differential speaker outputs capable of driving 16 Ω. Hardware 5-band
digital equalizer applied before DAC. Hardware FFT/FIR accelerator: 256-point
complex FFT or dual-channel 256-tap FIR filter.

### Temperature Sensor
Precision ±3 °C, range −40 to +125 °C. Result read via ADC channel.

### Wi-Fi
802.11b/g/n, HT20 and HT40. AP and STA simultaneously. TX active indicator
(GPIO25 / TXEN), RX active indicator (GPIO28 / RX_EN) for external PA/LNA
control.

### Bluetooth
BLE 4.2. Classic Bluetooth co-existence. Shares antenna with Wi-Fi via time
multiplexing. BT active indicator (GPIO8), packet priority indicator (GPIO9).
